3.2、字符串操作方法

  • concat()
  • slice()
  • substr()
  • substring()
3.2.1、cancat()

可以将一个或多个字符串拼接起来，返回一个新的字符串，不会改变原字符串。

但是在实际开发中使用更多的还是加号操作符（+），因为加号操作符方法更简单易行。

3.2.2、slice()

接收两个参数，分别是开始位置和结束位置，返回起始间位置的字符。当传入一个参数的时候，返回开始位置到字符串结束的字符。

3.2.3、substr()

接收两个参数，开始位置和向后取几个字符数，返回从开始位置到向后取几个字符的字符串。当传入一个参数的时候，返回从开始位置到字符串结束的字符。

3.2.4、substring()

他和 slice() 的返回结果是一样的。

如果给 slice()、substr() 和 substring() 传入负值，结果会是怎么样的？

slice() 和 substr() 如果只传入一个负数参数，相当于字符串 length 值+(负数参数)，也就是 12+(-3)=9，参数传入了 9，所以都返回了“ll!”。substring() 返回了全部字符串，也就是他把负值都转换成了 0。

当第二个参数传入负数的时候，slice() 方法会把第二个参数转换为 12+(-4)=8，所以 slice(3, 8) 返回的就是 “ is w”。substr() 方法会把第二个负数参数转换为 0，也就是 substr(3, 0)，从第三个位置取 0 个字符，也就返回了 0。substring() 方法也会把第二个负数参数转换为 0，也就是 substring(3, 0)，但是因为 substring() 截取的是开始到结束的位置，当开始位置小于结束位置时，就会自动将大的换为结束位置，所以就是 substring(0, 3)，返回的就是 “All”。

3.3、字符串位置方法

  • indexOf()
  • lastIndexOf()

向 indexOf() 传入一个字符参数的时候，他返回的就是这个字符在字符串中的位置。lastIndexOf() 与 indexOf() 基本类似，只不过他是从字符串右向左找，找到第一个符合参数的字符，返回他在字符串中的位置。indexOf() 和 lastIndexOf() 一样，如果没有找到该字符，则返回 -1。

3.6、字符串的模式匹配方法

String 类型提供了几个用于在字符串中匹配模式的方法：

  • match()
  • search()
  • replace()
  • split()
3.6.1、match()

只接收一个参数，要么是正则表达式，要么是一个 RegExp 对象，他返回一个数组。

3.6.2、search()

和 match() 方法一样，只接收一个参数，要么是正则表达式，要么是一个 RegExp 对象。他返回字符串中第一个匹配项的索引，如果没有找到匹配项，返回 -1。

根据指定的分割符将一个字符串分割成多个字符串，并放到一个数组中。分隔符可以是一个字符串，也可以是一个 RegExp 对象。可以接收可选的第二个参数，用于指定数组的大小。
