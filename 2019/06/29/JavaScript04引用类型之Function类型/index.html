4.2、this

this 引用的是函数据以执行的环境对象，在全局作用域中调用函数时，this 对象引用的就是 window。

上面代码 sayColor 是在全局作用域中定义的，他引用了 this 对象，由于在调用函数之前，this 的值并不确定，因此 this 可能会在代码执行过程中引用不同的对象，当在全局作用域中调用 sayColor 时，this 引用的是全局对象 window，也就是说 this.color 求值会转换为window.color，所以结果会返回“red”。
当把这个函数赋给 obj 并调用 obj.sayColor() 的时候，this 引用的是对象 obj，所以 this.color 的求值会转换为 obj.color，结果返回“blue”。

函数名仅仅是一个指针，即使在不同的环境中执行，全局的 sayColor 和 obj.sayColor 指向的任然是同一个函数。

五、函数属性和方法

ECMAScript 中函数是对象，所以函数也有属性和方法。

5.1、属性

每个函数都包含两个属性 length 和 prototype。

length 属性表示函数接收参数的个数：

prototype 属性是 ECMAScript 核心所定义的全部属性中，最耐人寻味的一个属性。对于引用类型而言，prototype 是保存他们所有实例方法的真正存在。也就是说，像 toString()、valueOf() 这些方法也都保存在 prototype 上，只不过是通过各自对象的实例访问罢了。

5.2、方法

每个函数都包含两个非继承而来的方法，apply() 和 call()，这两个方法都是在特定的作用域中调用函数，实际上等于设置函数体内 this 对象的值。

5.2.1、apply()

接收两个参数，第一个是在其中运行函数的作用域，第二个是参数数组，可以是 Array 实例，也可以是 arguments 对象：

applySum1 在执行 sum 函数的时候，传入了 this 作为 this 值（因为是在全局作用域中调用的，所以传入的就是 window 对象）和 arguments 对象。applySum2 同样也调用了 sum 函数，但他传入的是 this 和一个参数数组。这两个函数都会正常执行并返回正确的结果。

5.2.2、call()

call() 方法和 apply() 方法作用相同，只是接受参数的方式不同，对于 call() 方法，第一个参数 this 值没有变化，变化的是其余参数都直接传递个函数：

也就是说，在使用 call() 方法的时候，传递给函数的参数必须逐个列举出来。

apply() 方法和 call() 方法的结果没有什么不同，只是 call() 方法需要明确传入的每一个参数，而 apply() 方法只需要传入一个 arguments 对象或参数数组。在不给函数传参数的情况下，使用哪种方法都无所谓。

传递参数并非是 apply() 和 call() 的用武之地，他更强大的作用是能够扩充函数赖以运行的作用域：

在 call() 方法中，this 和 window 都指向的是 window，所以返回的都是 red，当运行 sayColor.call(obj) 的时候，函数的执行环境就不一样了，因为此时函数体内的 this 对象指向了 obj，所以结果显示 blue。
使用 apply() 和 call() 来扩充作用域最大的好处就是对象不需要与方法有任何耦合关系。
